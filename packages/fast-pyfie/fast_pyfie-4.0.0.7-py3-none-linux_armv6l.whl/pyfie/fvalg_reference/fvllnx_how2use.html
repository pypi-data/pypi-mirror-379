FIEライブラリの使い方(FVL/LNXユーザー向け)

提供バイナリ形態とライセンスについて
FIEライブラリはライセンス・提供形態の違いにより、幾つかの共有ライブラリに分けられて提供されます。
  • libfvalg.so
    弊社単独で作成しているライブラリであり、弊社のソフトウェア製品使用権許諾書の元でご使用頂けます。
  • libfvalg_oss.so
    弊社の作成部分に加えて、オープンソースソフトウェア を併せて利用したライブラリです。libfvalg_oss.so をご使用頂く場合は、 弊社のソフトウェア製品使用権許諾書に加えて、使用している各オープンソース ソフトウェアのライセンスの両方にご同意頂く必要がございます。 各オープンソースのライセンス条項は ＯＳＳライセンス条項 に記載されておりますので、 libfvalg_oss.so をご使用頂く場合は、良くお読み頂いた上、ライセンスを守ってご使用ください。
インクルードファイル
FIEライブラリの関数等の定義は fie.h にて定義されております。 ご使用の際は、使用するソースファイルにて fie.h をインクルードして ください。例外として、ヒープメモリ管理関数のみは oal_aloc.h にて 定義されております。こちらをご使用の際は oal_aloc.h を併せて インクルードしてご使用ください。
初期化
全てのFIEライブラリはライブラリ初期化関数 fnFIE_setup() が実行済み でなければなりません。 fnFIE_setup() が実行されていないと、正常に 動作しませんので、ユーザープログラム開始直後に fnFIE_setup() を実行 することをお忘れにならないようお願いいたします。
ユーザープログラム例
#include "fie.h"
#include "oal_aloc.h"

int main(void)
{
    // FIEライブラリの使用前に必ずコールする必要が有ります。
    fnFIE_setup();

    /* FIEライブラリを使用するコード */

    // 終了処理
    fnFIE_teardown();
    return 0;
}
FXcc IDE への設定
FXcc IDEの標準状態では、FIEライブラリはユーザーアプリケーションビルドのリンク対象に含まれておりません。 そのため、FIEライブラリを使用する場合にはFXcc IDEにてリンカの設定を行わなくてはいけません。
FXcc IDE のメニューから、「設定」->「コンパイラ、リンカ設定」をクリックすると、 下図の画面が表示されます。

カメラからの画像取り込み、画像の表示に関して
FIEライブラリでは、カメラからの画像取り込みや画像表示、パッド操作といった、画像処理以外のライブラリを用意していません。 そのため、カメラからの画像取り込みや処理画像の表示といった操作は、従来のFVL関数を使用してください。

  • グラフィックライブラリ

  • パッド関連ライブラリ
  • メニュー・バー関連ライブラリ
  • リスト（表形式）関連ライブラリ
  • フォトカプラ絶縁型I/O 拡張ライブラリ
  • システム制御ライブラリ
  • システムパラメータ更新／参照ライブラリ
カメラからの取り込み -> FIEライブラリを使用しての画像処理 -> 処理画像の表示までの手順につきましては、 FVL/LNXインストーラー付属のサンプルソフトに手順例が記載されていますので、ご参照ください。
カラー画像処理について
FIEライブラリでは、カラー画像に対する画像処理機能を用意しています(ex:色空間と色空間変換 , 色彩検査 )。 ですが、現状のFVL/LNXシステムでは、カラー画像の取り込みをすることができません。 対象ライブラリ使用の際にはご注意ください。
※バージョン 3.20/2.60以降はカラー画像を取り込めるようになりました。
